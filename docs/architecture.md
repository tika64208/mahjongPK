# 自动龙岩麻将系统架构设计

## 1. 架构原则

系统采用「规则核心 + 适配层」架构：

- 规则核心只负责麻将状态、动作校验、胡牌判断、机器人决策。
- 命令行、小程序、Web、App 都只是客户端适配层。
- 后续做多端时，将规则核心放到服务端，由各端通过 API 或 socket 与服务端交互。

## 2. 第一阶段模块

```text
longyan_mj/
  tiles.py       牌定义、排序、展示、牌墙构建
  evaluator.py   胡牌判断：普通胡、七对、十三幺、金牌
  scoring.py     胡牌倍数、庄闲支付、得分结果
  rules.py       桌规配置
  state.py       玩家、碰牌、明刻等状态对象
  shanten.py     向听与有效进张评估
  bot.py         机器人出牌与碰牌策略
  game.py        牌局流程、摸牌、出牌、碰牌、胜负
  cli.py         命令行交互入口
```

## 3. 核心流程

1. 创建规则配置。
2. 创建 4 名玩家，其中 0 号为真人，1-3 号为机器人。
3. 构建并打乱 136 张牌墙。
4. 翻出 1 张金牌指示牌，本张牌从牌墙移除。
5. 发牌，庄家 14 张，其余玩家 13 张。
6. 庄家先出牌。
7. 之后玩家按座次摸牌、判断自摸、出牌。
8. 其他玩家可按座次选择碰牌。
9. 有人胡牌则结束；牌墙摸完则流局。

## 4. 胡牌算法

胡牌判断分层实现：

- 三金倒：手牌中实际金牌数量达到 3 张。
- 七对：14 张牌可组成 7 个对子，金牌可补对子。
- 十三幺：满足 13 种幺九字牌加任意对子，金牌可补缺牌或对子。
- 普通胡：按「4 组面子 + 1 对将」递归拆牌，金牌可作为万能牌补缺。

对于已经碰出的玩家，普通胡牌只判断剩余暗手牌是否能组成「剩余面子 + 1 对将」。

## 5. 机器人能力分层

第一阶段提供两类机器人策略：

- `BasicBot`：基础策略，只根据对子、刻子、邻张、金牌保留价值出牌。
- `ShantenBot`：算牌策略，会分析每张候选出牌后的向听数；高手配置还会比较有效进张数量。

默认 3 个机器人能力不同：

- 机器人A：基础策略。
- 机器人B：向听策略。
- 机器人C：向听 + 有效进张策略。

机器人策略通过 `bot_policies` 注入到 `MahjongGame`，后续服务端可以按房间难度、玩家等级或付费配置切换不同 AI。

## 6. 计分设计

计分模块使用 `ScoreProfile` 保存底分、庄家倍数和每种胡牌的倍数表。当前 MVP 按自摸结算：

- 庄家自摸：三个闲家支付双倍。
- 闲家自摸：庄家支付双倍，其他闲家支付普通分。
- `ScoreResult` 返回四家分数变化，便于 CLI 展示，也便于后续服务端存储。

当前已实现普通自摸、七对、三金倒、十三幺；抢金、游金、双游、三游、七对游金类作为得分表预留项。

## 7. 多端演进设计

后续小程序或多端版本建议拆分为：

```text
clients/
  wechat-miniprogram/
  web/
  mobile/

server/
  api/
  websocket/
  persistence/
  bot_service/
  game_room/

core/
  rules_engine/
  evaluator/
  state_machine/
```

服务端应成为唯一可信规则执行者：

- 客户端只提交动作，例如 discard、pong、win。
- 服务端校验动作合法性并推进状态。
- 服务端广播增量事件，客户端只负责渲染。
- 每局保存 action log，便于断线恢复、复盘和纠错。

## 8. API 草案

后续服务端可提供：

- `POST /rooms` 创建房间。
- `POST /rooms/{room_id}/join` 加入房间。
- `POST /rooms/{room_id}/start` 开始牌局。
- `POST /games/{game_id}/actions` 提交动作。
- `GET /games/{game_id}` 获取当前牌局状态。
- `WS /games/{game_id}/events` 订阅牌局事件。

动作事件建议保持结构化：

```json
{
  "type": "discard",
  "player": 0,
  "tile": "M5",
  "turn": 18
}
```

## 9. 第一阶段限制

第一阶段实现的是可运行、可测试的规则引擎和命令行客户端，不处理联网、多端 UI、复杂结算、游金完整状态机。相关能力通过规则配置和状态机边界预留。
