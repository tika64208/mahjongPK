# 自动龙岩麻将系统需求分析

## 1. 项目目标

开发一个自动麻将系统，支持 1 名真人玩家与 3 个机器人进行龙岩麻将对局。第一阶段交付本地命令行 MVP，后续可扩展为小程序、多端 App 或联网对战版本。

## 2. 用户角色

- 真人玩家：通过客户端查看手牌、选择出牌、选择是否碰牌、选择是否胡牌。
- 机器人玩家：由系统自动摸牌、判断胡牌、决定是否碰牌和出牌。
- 系统：负责发牌、开金、回合推进、规则校验、胡牌判断、结算和对局日志。

## 3. 第一阶段功能范围

### 3.1 对局能力

- 支持 4 人局：1 名真人玩家 + 3 名机器人。
- 支持随机洗牌、发牌、开金。
- 支持按座次轮流摸牌、出牌。
- 支持不能吃牌。
- 支持碰牌。
- 支持暗杠、明杠、补杠和杠后补牌 MVP。
- 支持金牌作为万能牌参与胡牌判断。
- 支持自摸胡。
- 支持三金倒、抢金、七对、十三幺、普通胡牌型判断。
- 支持单游、双游 MVP 状态机和结算。
- 支持每种已实现胡牌类型的基础得分结算。
- 支持流局。
- 支持基础机器人策略。
- 支持不同机器人能力配置，包括基础策略、向听计算、有效进张计算。
- 支持专家机器人能力：剩余牌计算、胡牌价值评估、碰牌收益评估、防守风险评估、金牌策略。
- 支持机器人能力配置，为后续购买能力、账号权益和房间难度管理预留。
- 支持扩展能力配置：风格控制、对手听牌判断、危险牌精细评分、杠牌收益、游金策略、轻量模拟和出牌解释。

### 3.2 命令行交互

- 展示真人玩家手牌。
- 展示本局金牌。
- 展示每位玩家出牌、碰牌、杠牌、胡牌事件。
- 真人玩家可通过编号选择出牌。
- 真人玩家可选择是否碰牌、杠牌、是否自摸胡。

### 3.3 测试

- 胡牌算法单元测试。
- 向听和有效进张评估单元测试。
- 发牌与牌墙完整性测试。
- 碰牌、杠牌规则测试。
- 机器人出牌策略测试。

## 4. 第一阶段暂不实现

以下内容在架构中预留扩展点，但不作为第一阶段必需交付：

- 小程序 UI。
- 多端实时同步。
- 用户登录、房间匹配、好友房。
- 断线重连。
- 复杂账户、金币、积分榜和赛季结算。
- 抢杠胡。
- 三游的完整状态机。
- 游金相关抢胡、过胡、打金限制等完整桌规。
- 分饼。
- 花牌与花杠。
- 录像回放。
- 反作弊。

## 5. 后续小程序/多端需求

后续如果做成小程序或多端版，需要增加：

- 后端游戏服务：保存牌局状态，校验每个动作是否合法。
- 客户端适配层：微信小程序、Web、App 共用同一套接口。
- 实时通信：WebSocket 或小程序 socket，用于推送摸牌、出牌、碰牌、胡牌事件。
- 房间系统：创建房间、加入房间、准备、开始、解散。
- 玩家身份：登录、昵称、头像、战绩。
- 机器人服务：真人不足时补机器人，也可做单机陪练。
- 能力权益：账号可购买或启用不同机器人能力，服务端按权益生成对应机器人。
- 牌局持久化：保存每局动作日志，用于复盘和问题排查。
- 版本化规则配置：不同桌规可以用配置切换，而不是改代码。

## 6. 非功能需求

- 规则引擎必须与 UI 解耦。
- 胡牌判断必须可单元测试。
- 牌局动作必须可复现，支持固定随机种子。
- 核心模块尽量不依赖第三方库，降低后续迁移成本。
- 对局状态要能序列化，方便后续接入服务端和多端同步。
